# Graphic Novel Project Structure
To maintain consistency and streamline the production of your graphic novel, it is essential to keep a standardized directory structure. This ensures that assets like character sheets, scripts, and world-building notes are easily accessible as the project scales.

Graphic-Novel-Project-Name/
├── 01_Narrative_Core/
│   ├── Narrative_Synopsis.md
│   ├── World_Building.md
│   └── Timeline.md
├── 02_Characters/
│   ├── Leads/
│   ├── Supporting/
│   └── Concept_Art/
├── 03_Scripts/
│   ├── Issue_01_Script.md
│   └── Archival/
├── 04_Assets/
│   ├── Reference_Images/
│   ├── Textures/

├── 05_Production/
│   ├── Storyboards/
│   ├── Pencils_Inks/
│   └── Final_Renders/
└── README.md

## Core File Descriptions
01_Narrative_Synopsis.md: A high-level overview of the story arc. It should include the "Elevator Pitch," the primary conflict, and the resolution.

02_Characters/: Individual Markdown files for every major and minor character. This folder should also house a sub-folder for visual references or AI-generated character consistency sheets.

03_Scripts/: Full scripts broken down by page and panel. It is recommended to use a standardized format (e.g., Fountain or clear Markdown headers) to distinguish between dialogue, SFX, and panel descriptions.

04_Assets/: A repository for visual inspiration, historical references (e.g., 1970s architecture or military gear), and specific color palettes.
